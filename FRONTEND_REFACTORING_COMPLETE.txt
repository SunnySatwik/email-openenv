------

- Result: "Spam always shows Legitimate", "Priority always shows UNKNOWN", etc.

Solution:
---------
1. ✅ Validate API responses at client layer
2. ✅ Use optional chaining (?.) for safe nested access
3. ✅ Add fallback values with nullish coalescing (??)
4. ✅ Type-check before calling methods like .toFixed()
5. ✅ Show error messages when API fails

KEY CHANGES AT A GLANCE
==============================================================================

lib/api.js:
  OLD: return response.json();
  NEW: return normalizeResponse(response.json());
       ↓ Validates & ensures correct structure

ResultsCard.jsx:
  OLD: const isSpam = currentResult.action.is_spam;  // Can be undefined
  NEW: const isSpam = currentResult?.action?.is_spam ?? false;  // Always boolean

  OLD: {currentResult.reward.toFixed(2)}  // Crashes if undefined
  NEW: {typeof reward === "number" ? reward.toFixed(2) : "0.00"}  // Safe

app/page.jsx:
  OLD: setResult({ easy: results[0], medium: results[1], hard: results[2] });
  NEW: setResult({
         easy: results[0] || { action: {}, reward: 0, latency_ms: 0 },
         ...  // Fallback objects included
       });

==============================================================================
NULL-SAFETY PATTERNS USED
==============================================================================

Optional Chaining (?.):
  currentResult?.action?.is_spam      ← Returns undefined if any step is null

Nullish Coalescing (??):
  value ?? defaultValue               ← Uses default only if value is null/undefined
                                        (unlike || which treats 0, "", false as falsy)

Type Guards:
  typeof reward === "number"          ← Check before calling .toFixed()

Combined Pattern:
  currentResult?.action?.priority ?? "unknown"   ← Safe access + fallback

FALLBACK VALUES STRATEGY
==============================================================================

Field          | Fallback Value | Reason
---------------|----------------|-----------------------------------------
action         | {}             | Empty object prevents undefined errors

priority       | "unknown"      | Explicit for visibility
should_reply   | false          | Conservative: don't auto-reply
reply_text     | ""             | Empty string for missing text
reward         | 0              | Neutral reward
latency_ms     | 0              | Indicates error/missing data
